# Product Context: Demand Forecasting System

## Зачем Этот Проект Существует?

### Бизнес-Проблема
Розничные сети сталкиваются с критической проблемой:
- **Избыток товара** → заморозка капитала, издержки на хранение, списания
- **Дефицит товара** → потеря продаж, недовольство клиентов, репутационный ущерб

Традиционные методы прогнозирования (экспертные оценки, простые средние) дают большую ошибку и не учитывают сложные паттерны спроса.

### Что Решает Наша Система?
ML-система для точного прогнозирования спроса, которая:
1. **Анализирует исторические данные** о продажах
2. **Выявляет паттерны**: сезонность, тренды, аномалии
3. **Предсказывает будущий спрос** на горизонте 3 месяца
4. **Адаптируется** к изменениям в данных

### Ценность для Бизнеса
- **Финансовая**: снижение издержек на 10-20% через оптимизацию запасов
- **Операционная**: автоматизация процесса прогнозирования
- **Стратегическая**: данные для принятия решений о закупках и ассортименте

## Как Это Должно Работать?

### Основной Use Case
1. Пользователь запрашивает прогноз для конкретного магазина и товара
2. Система анализирует исторические данные и паттерны
3. Модель генерирует прогноз на следующие 90 дней
4. Результат возвращается в формате daily/weekly/monthly

### Дополнительные Use Cases
- **Batch-прогноз**: генерация прогнозов для всех 500 комбинаций магазин-товар
- **What-if анализ**: симуляция различных сценариев (промо-акции, праздники)
- **Мониторинг**: отслеживание точности прогнозов vs реальных продаж

## Пользовательский Опыт

### Для Data Scientist / Аналитика
- Jupyter notebooks для экспериментов
- Модульный код для быстрого прототипирования
- Визуализация результатов и метрик
- Reproducible pipeline

### Для Бизнес-Пользователя (Будущее)
- REST API для получения прогнозов
- Простой интерфейс: store_id + item_id → forecast
- Понятные метрики: confidence intervals, accuracy metrics
- Dashboard для визуализации (опционально)

## Целевые Метрики Качества

### ML Метрики
- **SMAPE < 15%**: конкурентоспособный уровень точности
- **Стабильность**: низкая вариативность между разными товарами/магазинами

### Системные Метрики
- **Latency**: < 100ms для single prediction
- **Throughput**: 500+ predictions/sec для batch
- **Availability**: 99.9% uptime (для продакшн-системы)

## Границы Проекта

### В Скоупе
- Прогнозирование на основе исторических продаж
- Feature engineering из временных данных
- Сравнение различных моделей
- API для inference

### Вне Скоупа (пока)
- Внешние факторы (погода, праздники, промо-акции)
- Реал-тайм обновление прогнозов
- Мультимодальные данные (изображения товаров, описания)
- Рекомендательная система
